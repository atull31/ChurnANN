# Customer Churn Prediction

A neural network model that predicts whether a bank customer is likely to churn (close their account), with an interactive Streamlit app for real-time predictions.

## Overview

This project uses an Artificial Neural Network (ANN) built with TensorFlow/Keras to perform binary classification on bank customer data. It covers the full workflow: data preprocessing, model training, evaluation, and deployment as a web app.

## Dataset

**`Churn_Modelling.csv`** — 10,000 bank customer records with the following features:

| Feature | Description |
|---|---|
| CreditScore | Customer's credit score |
| Geography | Country (France, Spain, Germany) |
| Gender | Male / Female |
| Age | Customer's age |
| Tenure | Years with the bank |
| Balance | Account balance |
| NumOfProducts | Number of bank products used |
| HasCrCard | Whether the customer has a credit card (0/1) |
| IsActiveMember | Whether the customer is an active member (0/1) |
| EstimatedSalary | Estimated annual salary |
| **Exited** | Target — whether the customer churned (0/1) |

`RowNumber`, `CustomerId`, and `Surname` are dropped during preprocessing as they carry no predictive value.

## Model Architecture

A simple feed-forward ANN built with Keras' `Sequential` API:

| Layer | Units | Activation |
|---|---|---|
| Dense (hidden 1) | 64 | ReLU |
| Dense (hidden 2) | 32 | ReLU |
| Dense (output) | 1 | Sigmoid |

- **Optimizer:** Adam (learning rate = 0.01)
- **Loss:** Binary Cross-Entropy
- **Callbacks:** `EarlyStopping` (patience = 10, restores best weights) and `TensorBoard` for training visualization

### Preprocessing
- `Gender` — encoded with `LabelEncoder`
- `Geography` — encoded with `OneHotEncoder`
- All features — scaled with `StandardScaler`
- Train/test split — 80/20
